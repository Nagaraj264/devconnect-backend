# Resume DevConnect Frontend Build

// turbo-all

When the user wants to continue working on the DevConnect frontend project, follow these steps:

## 1. Read the Progress Tracker
Read `frontend/PROGRESS.md` to understand:
- What phase and step we are currently on
- What has already been completed
- What the immediate next steps are

## 2. Read the Roadmap
Read `frontend/ROADMAP.md` to understand:
- The full learning plan with 8 phases
- What React concepts are taught in each phase
- Which Stitch reference files correspond to each step

## 3. Check Stitch References
The `frontend/stitch/` folder contains HTML files exported from Google Stitch.
Each subfolder has a `code.html` (the HTML/Tailwind code) and `screen.png` (the design screenshot).
These are the visual targets we are building toward.

## 4. Check Current Code State
Look at what files exist in `frontend/src/components/` and `frontend/src/pages/` to understand what has already been built.

## 5. Resume Teaching
- Pick up from the exact step listed in PROGRESS.md under "Currently Working On"
- Explain every concept before writing code
- Let the user write the code themselves — guide, don't do it for them
- After completing a step, update PROGRESS.md to reflect the new status

## 6. End-of-Session Checklist
Before ending any session, always:
- Update `frontend/PROGRESS.md` with what was completed
- Move the "Currently Working On" section to the next step
- Commit a summary of what was learned
